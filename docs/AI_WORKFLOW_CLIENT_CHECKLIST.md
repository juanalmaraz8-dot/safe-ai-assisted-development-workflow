# AI Workflow Client Checklist
## Purpose
Use this checklist before doing AI-assisted work on any client, portfolio, or real project repo.
## 1. Intake
- What is the project?
- What problem should be improved?
- What files are allowed to change?
- What files are blocked?
- What counts as done?
- What command tests the project?
## 2. Safety baseline
- Confirm repo is under git.
- Confirm git status is clean.
- Confirm test command runs before changes.
- Confirm backup/checkpoint exists.
## 3. Plan before edit
- Inspect files first.
- Create short implementation plan.
- Reject plans that invent facts, modify too much, or bypass tests.
## 4. Sandbox work
- Edit only in sandbox.
- Run sandbox test.
- Generate visible diff.
- For new files, make content visible before approval.
## 5. Approval gate
- Review diff.
- Check additions/deletions/files changed.
- Reject destructive or unrelated changes.
- Approve only scoped changes.
## 6. Source apply
- Apply approved patch only.
- Run source test.
- Commit with clear message.
- Confirm git clean.
## 7. Delivery
- Summarize what changed.
- Include test result.
- Include commit hash.
- Include before/after notes.
- Save checkpoint.
## 8. Money alignment
Work is valuable only if it improves at least one:
- income potential
- portfolio value
- reusable system
- marketable skill
- operational efficiency
- long-term project value
