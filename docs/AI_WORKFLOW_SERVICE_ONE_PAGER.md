# Safe AI-Assisted Development Workflow
## What I help with
I help small projects use AI for code and documentation work without losing control of the repo.
## The problem
AI coding tools can be useful, but they often create risk:
- unclear changes
- too many edits at once
- skipped tests
- invented details
- weak review process
- poor project history
## The workflow
My workflow keeps AI-assisted work controlled:
1. Inspect the project.
2. Create a plan before editing.
3. Edit only in a sandbox.
4. Run tests before applying changes.
5. Show the diff clearly.
6. Require approval before source changes.
7. Apply only approved changes.
8. Run source tests.
9. Commit with git.
10. Save a checkpoint.
## Proof
I used this workflow to improve a real Godot project, Ancient Machine Ruins, from a basic prototype into a documented first-playable portfolio artifact.
## What this proves
- safer AI-assisted development
- better documentation
- clearer repo history
- test-backed changes
- human approval preserved
- recoverable workflow through checkpoints
## Best fit
This workflow is useful for:
- solo developers
- small teams
- creators with messy project folders
- students building portfolio projects
- small businesses needing automation or documentation
## Simple offer
I can inspect a small repo or project folder, identify a safe next improvement, make changes in a sandbox, show the diff, run tests, and leave behind documentation plus a clean project history.
